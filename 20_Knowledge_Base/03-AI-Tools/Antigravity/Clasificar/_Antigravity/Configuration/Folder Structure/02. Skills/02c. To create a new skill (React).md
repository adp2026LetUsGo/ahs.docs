To create a new skill in Google Antigravity, follow this boilerplate structure. For this example, we’ll build a **`react-component-generator`** skill.

1. Folder Setup

Create a dedicated folder for your skill within the workspace directory: 

bash

```
mkdir -p .agent/skills/react-component-generator/scripts
mkdir -p .agent/skills/react-component-generator/resources
```

Use code with caution.

2. The `SKILL.md` File

Create `.agent/skills/react-component-generator/SKILL.md`. This file must follow a specific pattern for the agent to recognize it: 

markdown

```
---
name: react-component-generator
description: Generates standardized React functional components with TypeScript and CSS modules. Use when the user asks to "create a component" or "scaffold a UI element."
---

# React Component Generator
When this skill is activated, follow these steps strictly:

1. **Naming**: Use PascalCase for the component name (e.g., `MyButton`).
2. **Template**: Read the base structure from `./resources/component-template.txt`.
3. **Execution**: 
   - Generate the `.tsx` file in the requested directory.
   - Run the script `./scripts/create_css_module.py <ComponentName>` to generate the accompanying `.module.css`.
4. **Imports**: Ensure all components use the project-standard `import React from 'react';`.
```

Use code with caution.

3. Adding a Resource (Optional)

Create `.agent/skills/react-component-generator/resources/component-template.txt` to give the agent a "Gold Standard" to copy:

typescript

```
import React from 'react';
import styles from './{{NAME}}.module.css';

interface {{NAME}}Props {
  children?: React.ReactNode;
}

export const {{NAME}}: React.FC<{{NAME}}Props> = ({ children }) => {
  return <div className={styles.container}>{children}</div>;
};
```

Use code with caution.

4. Adding a Script (Optional)

Create a simple Python script at `.agent/skills/react-component-generator/scripts/create_css_module.py` to handle file system side-effects:

python

```
import sys
import os

component_name = sys.argv[1]
filename = f"{component_name}.module.css"

with open(filename, "w") as f:
    f.write(".container {\n  display: block;\n}")

print(f"Successfully created {filename}")
```

Use code with caution.

Best Practices for Custom Skills

- **Atomic Scopes**: Create separate skills for different workflows (e.g., one for testing, one for documentation) rather than one giant "All-In-One" skill.
- **Third-Person Descriptions**: Write the `description` in the third person (e.g., "Analyzes logs...") as this is injected into the system prompt.
- **Immediate Feedback**: If a script fails, instruct the agent in `SKILL.md` to report the specific error to you.
