El **User Task Analysis** describe **qué hace el usuario paso a paso** para lograr un objetivo.

- Se centra en:
    
    - Acciones del usuario
        
    - Decisiones
        
    - Flujo mental y operativo
        
- No habla del sistema internamente, solo de la interacción desde el punto de vista humano
    

✅ Ejemplo:

> “El usuario inicia sesión → busca producto → añade al carrito → paga”

📌 Se usa para:

- UX/UI design
    
- Entender necesidades reales
    
- Detectar fricciones

El **User Interaction Flow** describe **cómo el usuario navega dentro del sistema** (pantallas, estados, decisiones).

- Se centra en:
    
    - Pantallas (UI)
        
    - Transiciones
        
    - Eventos (clicks, inputs)
        

✅ Ejemplo:

> Login Screen → Dashboard → Product Page → Checkout → Confirmation

📌 Diferencia clave con UTA:

- UTA = qué hace el usuario
    
- Interaction Flow = **dónde lo hace dentro del sistema**
    

📌 Se usa para:

- Diseño de interfaces
    
- Prototipos
    
- Arquitectura de navegación

6

El **System Sequence Specification** (muy cercano a un _System Sequence Diagram_) describe:

👉 **qué mensajes intercambian usuario y sistema en orden temporal**

- Se centra en:
    
    - Requests
        
    - Responses
        
    - Lógica del sistema
        
- Es más técnico
    

✅ Ejemplo:

Usuario → Sistema: login(email, password)  
Sistema → DB: validateUser()  
Sistema → Usuario: token

📌 Se usa para:

- Diseño técnico
    
- Definir APIs
    
- UML (diagramas de secuencia)
    

---

# ⚖️ Comparación rápida

|Aspecto|User Task Analysis|User Interaction Flow|System Sequence Specification|
|---|---|---|---|
|Enfoque|Usuario|Usuario + UI|Sistema|
|Nivel|Conceptual|Interfaz|Técnico|
|Describe|Qué hace el usuario|Cómo navega|Qué hace el sistema|
|Elementos|Tareas|Pantallas|Mensajes|
|Uso|UX research|UI design|Arquitectura|

---

# 🧠 Cómo se conectan (muy importante)

Piensa en esto como capas:

1. **User Task Analysis**  
    → Define el comportamiento humano
    
2. **User Interaction Flow**  
    → Lo traduce en pantallas y navegación
    
3. **System Sequence Specification**  
    → Lo convierte en lógica técnica y comunicación interna
    

---

# 🚀 Resumen simple

- 🧑 UTA → _Qué quiere hacer el usuario_
    
- 📱 Interaction Flow → _Cómo lo hace en la app_
    
- ⚙️ SSS → _Cómo lo procesa el sistema_
