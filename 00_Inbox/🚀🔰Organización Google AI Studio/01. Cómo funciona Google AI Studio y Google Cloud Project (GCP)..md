## 1. La Jerarquía de Organización

En AI Studio, la estructura no es exactamente "Un Proyecto = Muchas Apps", sino que depende de la integración con **Google Cloud Project (GCP)**.

- **Google Cloud Project (El contenedor raíz):** Cada vez que creas algo en AI Studio, está vinculado a un proyecto de Google Cloud.
    
- **API Keys:** Las API Keys pertenecen a un proyecto de GCP específico. Puedes generar varias llaves dentro de un mismo proyecto, pero todas compartirán los límites de cuota de ese proyecto.
    
- **Apps / Implementaciones:** En el contexto de AI Studio, lo que tú llamas "Apps" son generalmente los **Prompts guardados** (Chat prompts o Structured prompts).
    

### Relación de Límites

- **Límite de Proyectos:** Efectivamente, existe un límite de proyectos gratuitos (usualmente alrededor de 10-12) que puedes crear directamente desde la interfaz de AI Studio antes de que te pida gestionarlos desde la consola de Google Cloud.
    
- **Cuotas (Rate Limits):** El límite de "RPM" (peticiones por minuto) se aplica por **modelo** dentro de un **proyecto**. Si tienes dos "apps" usando la misma API Key, ambas compiten por la misma velocidad de procesamiento.
